Create storage pool

On this page, we are going to create a storage pool on the ceph cluster and mount it on the client. We are using the utility VM as client to test and verify the ceph storage cluster functionality.

  1. Login to utility VM as root user and execute below commands to install ceph-common package for client side utilities.

    mv /etc/yum.repos.d/rhel_dvd.repo /tmp/
rpm -e katello-ca-consumer-satellite-dle.ole.redhat.com
subscription-manager register
subscription-manager repos --disable=*
subscription-manager repos --enable=rhel-9-for-x86_64-baseos-rpms --enable=rhel-9-for-x86_64-appstream-rpms --enable=rhceph-7-tools-for-rhel-9-x86_64-rpms
dnf -y install chrony ceph-common

  2. Login to cephservera VM as root user.

    ssh root@cephservera

  3. Create a test storage pool on your cluster.

    ceph osd pool create test_pool 32 32

    The above command creates a replicated pool called test_pool with 32 placement groups.

    Sample Output
    pool 'test_pool' created

  4. Initialize the pool.

    rbd pool init test_pool

    Above command sets the application type for the pool to rbd.

  5. List the pools to verify test_pool is created.

    ceph osd lspools
    Sample Output
    1 .mgr
2 .rgw.root
3 default.rgw.log
4 default.rgw.control
5 default.rgw.meta
6 test_pool

  6. Check output of ceph df command to list the configured pools and view the usage and availability for test_pool.

    ceph df
    Sample Output
    --- RAW STORAGE ---
CLASS     SIZE    AVAIL     USED  RAW USED  %RAW USED
hdd    200 GiB  199 GiB  1.3 GiB   1.3 GiB       0.66
TOTAL  200 GiB  199 GiB  1.3 GiB   1.3 GiB       0.66

--- POOLS ---
POOL                 ID  PGS   STORED  OBJECTS     USED  %USED  MAX AVAIL
.mgr                  1    1  598 KiB        2  1.8 MiB      0     63 GiB
.rgw.root             2   32  1.4 KiB        4   48 KiB      0     63 GiB
default.rgw.log       3   32  3.7 KiB      177  412 KiB      0     63 GiB
default.rgw.control   4   32      0 B        8      0 B      0     63 GiB
default.rgw.meta      5   32    382 B        2   24 KiB      0     63 GiB
test_pool             6   32     19 B        1   12 KiB      0     63 GiB

    The ID of test_pool might be different in your lab environment.

  7. Create a dedicated Ceph user (client.test_pool.utility) for the client.

    ceph auth get-or-create client.test_pool.utility mon 'profile rbd' osd 'profile rbd' -o /etc/ceph/ceph.client.test_pool.utility.keyring
    We are using the utility VM as client to test and verify the ceph storage cluster deployment.

  8. Display the new file created.

    cat /etc/ceph/ceph.client.test_pool.utility.keyring

  9. Verify that the client.test_pool.utility user now exists.

    ceph auth get client.test_pool.utility

  10. Copy the Ceph configuration and the key-ring files from the /etc/ceph/ directory on the cephservera node to the /etc/ceph/ directory on the utility node.

    scp  /etc/ceph/{ceph.conf,ceph.client.test_pool.utility.keyring} root@utility:/etc/ceph/

  11. Login to the client system and temporarily set the default user ID used for connections to the cluster to test_pool.utility.

    export CEPH_ARGS='--id=test_pool.utility'

  12. Verify that you can connect to the cluster.

    rbd ls test_pool

  13. Create a new RADOS Block Device (RBD) image of size 128 megabytes.

    rbd create test_pool/test --size=128M

  14. Verify the image is successfully created under the pool.

    rbd ls test_pool

  15. Verify the parameters of the RBD image

    rbd info test_pool/test

  16. Map the RBD image on the utility node by using the kernel RBD client.

    rbd map test_pool/test

  17. Verify that you can see the RBD image mapped on the utility node like a regular disk block device.

    rbd showmapped
lsblk

  18. Format the device with an XFS file system

    mkfs.xfs /dev/rbd0

  19. Create a mount point for the file system and mount the file system created on the /dev/rbd0 device.

    mkdir /mnt/rbd
mount /dev/rbd0 /mnt/rbd

  20. Review the file-system usage.

    df /mnt/rbd

  21. Add some content to the file system.

    dd if=/dev/zero of=/mnt/rbd/test1 bs=10M count=1

  22. Review the file-system usage.

    df /mnt/rbd

  23. Review the content of the cluster.

    ceph df

  24. Unmount the file system and unmap the RBD image on the utility node.

    umount /mnt/rbd
rbd unmap /dev/rbd0
rbd showmapped
